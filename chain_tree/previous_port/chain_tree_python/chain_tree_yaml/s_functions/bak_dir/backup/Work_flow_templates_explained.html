📋 Workflow Templates for LispSequencer

⚠️ Important: Save the template code below as workflow_templates.mako to use with LispSequencer.

📄 Complete Template File

<%doc> Workflow-specific macros for common patterns in LispSequencer </%doc> <%def name="retry_block(branch_id, max_retries)"> (@CFL_FORK ${branch_id}) (@SET_RETRY_COUNT ${max_retries}) (@LOG "Starting retry block ${branch_id} with max retries: ${max_retries}") (!CFL_JOIN ${branch_id}) </%def> <%def name="timeout_block(branch_id, timeout_seconds)"> (@CFL_FORK ${branch_id}) (!CFL_WAIT ${timeout_seconds}) (?TIMEOUT_CHECK ${branch_id}) (@CFL_TERMINATE ${branch_id}) </%def> <%def name="error_handler(branch_id, error_message)"> (@CFL_FORK ${branch_id}) (@LOG_ERROR "${error_message}") (@SEND_NOTIFICATION "error" "${error_message}") (!CFL_JOIN ${branch_id}) </%def> <%def name="sequential_steps(*step_names)"> % for i, step in enumerate(step_names): (@CFL_FORK ${i}) (@EXECUTE_STEP "${step}") (!CFL_JOIN ${i}) % endfor </%def>

🔧 Template Reference

1. retry_block

Creates a retry pattern with configurable retry count.

Parameter Type Description
branch_id integer Branch identifier
max_retries integer Maximum number of retry attempts
Usage:
<%retry_block(0, 3)%>
Expands to:
(@CFL_FORK 0)
(@SET_RETRY_COUNT 3)
(@LOG "Starting retry block 0 with max retries: 3")
(!CFL_JOIN 0)
2. timeout_block

Creates a timeout check pattern.

Parameter Type Description
branch_id integer Branch identifier
timeout_seconds integer Timeout duration in seconds
Usage:
<%timeout_block(1, 30)%>
Expands to:
(@CFL_FORK 1)
(!CFL_WAIT 30)
(?TIMEOUT_CHECK 1)
(@CFL_TERMINATE 1)
3. error_handler

Creates an error handling pattern with logging and notification.

Parameter Type Description
branch_id integer Branch identifier
error_message string Error message to log and notify
Usage:
<%error_handler(2, "Database connection failed")%>
Expands to:
(@CFL_FORK 2)
(@LOG_ERROR "Database connection failed")
(@SEND_NOTIFICATION "error" "Database connection failed")
(!CFL_JOIN 2)
4. sequential_steps

Processes multiple steps sequentially with fork/join for each.

Parameter Type Description
*step_names strings Variable number of step names
Usage:
<%sequential_steps("validate", "transform", "load")%>
Expands to:
(@CFL_FORK 0)
(@EXECUTE_STEP "validate")
(!CFL_JOIN 0)
(@CFL_FORK 1)
(@EXECUTE_STEP "transform")
(!CFL_JOIN 1)
(@CFL_FORK 2)
(@EXECUTE_STEP "load")
(!CFL_JOIN 2)

💡 Complete Example

Python Code:

from lisp_sequencer import LispSequencer # Initialize with workflow templates sequencer = LispSequencer( handle={"session": "abc123"}, run_function=my_executor, user_macro_file='workflow_templates.mako' ) # Use the templates instruction = """ (pipeline (@CFL_KILL_CHILDREN) <%retry_block(0, 5)%> <%timeout_block(1, 60)%> <%sequential_steps("extract", "process", "save")%> <%error_handler(99, "Workflow failed")%> 'CFL_FUNCTION_TERMINATE) """ result = sequencer.run(instruction)

📚 Loading Templates

Method 1: At Initialization

sequencer = LispSequencer( handle=handle, run_function=func, user_macro_file='workflow_templates.mako' )

Method 2: After Initialization

sequencer.set_user_macros_from_file('workflow_templates.mako')

Method 3: Temporarily for One Call

sequencer.run(instruction, additional_macro_files='workflow_templates.mako')

Method 4: From String

template_string = """ <%def name="my_macro(param)"> (@SOME_FUNCTION "${param}") </%def> """ sequencer.set_user_macros_from_string(template_string)

✍️ Creating Your Own Templates

Basic Template:

<%def name="your_macro_name(param1, param2)"> (@YOUR_FUNCTION "${param1}") (@ANOTHER_FUNCTION ${param2}) </%def>

With Loops:

<%def name="batch_process(*items)"> % for i, item in enumerate(items): (@PROCESS ${i} "${item}") % endfor </%def>

With Conditionals:

<%def name="conditional_step(condition, action)"> % if condition: (@EXECUTE "${action}") % else: (@SKIP "${action}") % endif </%def>
